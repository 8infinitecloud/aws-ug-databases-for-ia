# Contributing to Financial Compliance RAG Lab

Thank you for your interest in contributing to this project! This is an AWS Community lab for demonstrating RAG and memory patterns with Amazon Bedrock.

## Code of Conduct

This project has adopted the [Amazon Open Source Code of Conduct](https://aws.github.io/code-of-conduct). See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for more information.

## Security Issue Notifications

If you discover a potential security issue, please **do NOT create a public GitHub issue**. Follow our [security policy](https://aws.amazon.com/security/vulnerability-reporting/) and email AWS Security at aws-security@amazon.com.

## How to Contribute

### Reporting Bugs

Open an issue with:
- Clear description of the bug
- Steps to reproduce
- Expected vs actual behavior
- AWS region and service versions used

### Suggesting Enhancements

- Open an issue describing the enhancement
- Explain the use case and why it would benefit lab participants

### Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-enhancement`
3. Make your changes following the conventions below
4. Test locally with the provided sample data
5. Submit a PR with a clear description

### Development Conventions

- **Python**: Follow PEP 8, use type hints, Black formatter
- **Comments**: Explain the *why* (decisions, trade-offs), not the *what*
- **ADRs**: Any architectural change should update `DECISIONS.md`
- **Costs**: If adding new AWS services, update `COST_ESTIMATE.md`
